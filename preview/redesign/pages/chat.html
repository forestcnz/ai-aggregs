用户问的是网关作用和三种协议差异。需要从聚合、转换、故障转移三点切入，再对比 Chat / Responses / Anthropic 的请求结构。
ai-aggregs 网关在本地起一个 HTTP 端点，把你的请求按顺序转发到多个上游提供商，并在 429 / 5xx / 超时时自动切换到下一个提供商或下一把密钥（故障转移）。 三种协议的区别： • Chat — OpenAI 经典 messages 结构，/v1/chat/completions • Responses — OpenAI 新版 input 结构，/v1/responses • Anthropic — messages + max_tokens，/v1/messages，用 x-api-key 鉴权 网关会在三种协议间做转换，所以你可以用任意一种协议消费，后端提供商可以是任意协议。
