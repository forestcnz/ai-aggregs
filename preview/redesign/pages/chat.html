用三句话解释什么是反向代理，以及它在 API 聚合中的作用
AI
THINKING
用户想要简洁的解释。反向代理是核心概念，应聚焦它如何隐藏后端、聚合多个上游。在 API 聚合场景中，关键点是统一入口 + 协议转换 + 故障转移。
反向代理是位于客户端与后端服务器之间的中间层，它接收所有外部请求再按规则转发到真正的后端，对外只暴露统一端点而隐藏后端拓扑。 在 API 聚合中，反向代理把多个上游提供商（OpenAI、Anthropic、GLM…）汇聚到单一本地端点，按请求路径自动判定协议并互转，配合密钥池与故障转移实现高可用。
