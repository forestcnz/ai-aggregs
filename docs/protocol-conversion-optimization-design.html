四、详细设计

4.1 reasoning_bridge envelope（核心创新）

4.1.1 问题精确描述

当前 ai-aggregs 在 Anthropic → Responses 方向（经 Chat 双跳）：

Anthropic 请求 → anthropic_to_chat_req → Chat 请求 → chat_to_responses_req → Responses 请求

anthropic_to_chat_req 提取 Anthropic thinking 块为 reasoning_content + reasoning_signature（非标准 Chat 扩展，converter.rs:370-382）。但 chat_to_responses_req 不识别 reasoning_signature，转换时丢失，最终 Responses 请求中 reasoning 历史不完整。

4.1.2 解决方案

借鉴 cc-switch reasoning_bridge.rs:11，用版本化 base64 envelope 把对方协议的完整 reasoning 上下文编码为字符串，作为 Chat reasoning_signature 字段值透传。

4.1.3 envelope 设计

新增文件：src-tauri/src/gateway/reasoning_bridge.rs（约 150 行）

// 双向版本化前缀
pub(crate) const ANTHROPIC_THINKING_ENVELOPE_PREFIX: &str = "aiaggregs-anthropic-thinking-v1:";
pub(crate) const OPENAI_REASONING_ENVELOPE_PREFIX: &str = "aiaggregs-openai-reasoning-v1:";

// 把 Anthropic thinking 块（含 signature）编码为 envelope 字符串
pub(crate) fn encode_anthropic_thinking(block: &Value) -> Option<String> {
    if block.get("type").and_then(|v| v.as_str()) != Some("thinking") {
        return None;
    }
    let sig = block.get("signature").and_then(|v| v.as_str()).unwrap_or("");
    if sig.is_empty() { return None; }
    // 防止嵌套编码：如果 sig 本身已是 envelope，原样返回
    if is_envelope(sig) { return Some(sig.to_string()); }
    let bytes = serde_json::to_vec(block).ok()?;
    Some(format!("{ANTHROPIC_THINKING_ENVELOPE_PREFIX}{}", URL_SAFE_NO_PAD::encode(bytes)))
}

pub(crate) fn decode_envelope(envelope: &str) -> Option<Value> {
    let payload = envelope.strip_prefix(ANTHROPIC_THINKING_ENVELOPE_PREFIX)
        .or_else(|| envelope.strip_prefix(OPENAI_REASONING_ENVELOPE_PREFIX))?;
    let bytes = URL_SAFE_NO_PAD::decode(payload).ok()?;
    serde_json::from_slice::<Value>(&bytes).ok()
}

pub(crate) fn is_envelope(s: &str) -> bool {
    s.starts_with(ANTHROPIC_THINKING_ENVELOPE_PREFIX)
        || s.starts_with(OPENAI_REASONING_ENVELOPE_PREFIX)
}

关键安全保证

  1. 向后兼容：当 reasoning_signature 不是 envelope 前缀时，decode_envelope 返回 None，回退到原逻辑
  2. 跨进程兼容：旧版客户端发的非 envelope signature 仍能透传
  3. 跨版本兼容：若 envelope 格式变化，前缀 v1 升级为 v2，旧版识别失败时回退
  4. 防嵌套：encode_anthropic_thinking 检测已是 envelope 时不再二次编码

4.2 tool_call 乱序兜底

4.2.1 当前问题

ChatToAnthropicStream 假设上游按 id → name → arguments 顺序到达（stream.rs:715-738）。DeepSeek/GLM/Zhipu 等上游有时先发 id+arguments 后发 name，导致 content_block_start.tool_use 用空 name 发出。

4.3 Claude Code billing header 过滤

Claude Code 在 system 字段开头注入动态 x-anthropic-billing-header:，破坏上游 cache prefix 匹配。借鉴 cc-switch transform.rs:18。

新增模块：gateway/converter/billing.rs（约 60 行，直接复制 cc-switch 实现 + 4 个测试）

const ANTHROPIC_BILLING_HEADER_PREFIX: &str = "x-anthropic-billing-header:";

pub(crate) fn strip_leading_anthropic_billing_header(text: &str) -> &str {
    if !text.starts_with(ANTHROPIC_BILLING_HEADER_PREFIX) { return text; }
    let Some(line_end) = text.as_bytes().iter().position(|b| *b == b'\n' || *b == b'\r') else { return ""; };
    let bytes = text.as_bytes();
    let mut rest_start = line_end + 1;
    if bytes[line_end] == b'\r' && bytes.get(line_end + 1) == Some(&b'\n') { rest_start += 1; }
    let rest = &text[rest_start..];
    if let Some(s) = rest.strip_prefix("\r\n") { s }
    else if let Some(s) = rest.strip_prefix('\n') { s }
    else { rest }
}

调用点：所有从 Anthropic 提取 system 的转换函数（chat_to_anthropic_req、anthropic_to_chat_req、chat_to_responses_req）。

4.4 JSON Schema normalization

OpenAI Responses / 严格 Chat 上游要求 tool parameters 必须是 type: object 且含 properties。借鉴 cc-switch transform.rs:494。

新增模块：gateway/converter/normalize.rs（约 80 行 + 4 测试）

pub(crate) fn clean_schema(schema: Value) -> Value {
    clean_schema_inner(schema, true)
}

fn clean_schema_inner(mut schema: Value, is_root: bool) -> Value {
    if let Some(obj) = schema.as_object_mut() {
        let missing_type = is_root && !obj.contains_key("type");
        if missing_type { obj.insert("type".into(), json!("object")); }
        if missing_type && !obj.contains_key("properties") {
            obj.insert("properties".into(), json!({}));
        }
        if obj.get("format").and_then(|v| v.as_str()) == Some("uri") {
            obj.remove("format");
        }
        // 递归
    }
    schema
}

调用点：所有 tools 转换处。

4.5 cache_control 字段剥离

Anthropic 协议的 cache_control: {type: "ephemeral"} 在转到 Chat / Responses 时，部分严格上游（GLM/Qwen）400。借鉴 cc-switch 回归测试 test_regression_gh3805。

// gateway/converter/fields.rs
pub(crate) fn strip_cache_control(value: &mut Value) {
    match value {
        Value::Object(obj) => {
            obj.remove("cache_control");
            for v in obj.values_mut() { strip_cache_control(v); }
        }
        Value::Array(arr) => {
            for v in arr { strip_cache_control(v); }
        }
        _ => {}
    }
}

调用点：仅跨协议方向（Anthropic → Chat/Responses）调用；Anthropic → Anthropic 透传不调用。

4.6 cache token 多字段兼容

当前 extract_usage（stream.rs:59）仅识别 Anthropic 标准字段。修改后兼容 OpenAI nested 字段：

fn extract_cache_field(usage: &Value, top_fields: &[&str], nested: &[(&str, &str)]) -> u64 {
    // 顶层直接字段优先
    for field in top_fields {
        if let Some(v) = usage.get(field).and_then(|x| x.as_u64()) { return v; }
    }
    // nested 路径 fallback
    for (parent, child) in nested {
        if let Some(v) = usage.pointer(&format!("/{parent}/{child}")).and_then(|x| x.as_u64()) {
            return v;
        }
    }
    0
}

pub fn extract_usage(v: &Value) -> Option<(u64, u64, u64)> {

                {"type": "tool_use", "id": "toolu_1", "name": "weather", "input": {"city": "SF"}}
            ]
        }]
    });
    // Anthropic → Chat → Responses → Chat → Anthropic（完整双跳）
    let chat = anthropic_to_chat_req(&original);
    let responses = chat_to_responses_req(&chat);
    let chat_back = responses_to_chat_req(&responses);
    let back = chat_to_anthropic_req(&chat_back);
    // thinking.signature 必须能往返（通过 reasoning_bridge envelope）
    assert_eq!(
        back["messages"][0]["content"][0]["signature"],
        original["messages"][0]["content"][0]["signature"]
    );
}
